# vbook 项目概述

> 版本: v0.1.0 | 最后更新: 2026-03-23 | 状态: MVP

---

## 项目愿景

**vbook (video notebook)** 旨在将视频内容转换为结构化的知识文档，帮助用户高效地从视频中提取、整理和复用知识。

### 核心使命

让视频知识可搜索、可复用、可传播。

---

## 为什么需要 vbook？

### 问题背景

在线教育、技术分享、会议录像等视频内容爆炸式增长，但视频作为知识载体存在明显缺陷：

| 问题 | 影响 |
|------|------|
| **不可搜索** | 无法快速定位关键信息 |
| **不可复用** | 难以引用、摘录、整理 |
| **学习效率低** | 必须线性观看，无法跳读 |
| **知识沉淀难** | 看过即忘，难以形成笔记 |

### vbook 的解决方案

将视频转换为**图文并茂的 Markdown 文档**，包含：
- 📝 **结构化大纲** - 自动提取知识层次
- ⏱️ **时间戳标注** - 快速定位原视频位置
- 🖼️ **关键截图** - 保留重要的视觉信息
- 🔍 **关键词提取** - 便于搜索和索引

## 核心功能

### 当前功能（v0.1.0 MVP）

```
┌─────────────────────────────────────────────────────────┐
│                    vbook Pipeline                        │
└─────────────────────────────────────────────────────────┘
                          │
        ┌─────────────────┼─────────────────┐
        │                 │                 │
        ▼                 ▼                 ▼
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│ 音频提取      │  │ 语音转录      │  │ 内容分析      │
│ FFmpeg       │→ │ Whisper      │→ │ LLM          │
│              │  │ (带时间戳)    │  │ (生成大纲)    │
└──────────────┘  └──────────────┘  └──────────────┘
                                            │
                          ┌─────────────────┘
                          │
                          ▼
                  ┌──────────────┐  ┌──────────────┐
                  │ 截图提取      │→ │ 文档生成      │
                  │ 关键帧       │  │ Markdown     │
                  └──────────────┘  └──────────────┘
```

#### 1. 音频提取（AudioExtractStage）
- 使用 FFmpeg 从视频中提取音频
- 输出格式：16-bit PCM, 16kHz 单声道 WAV
- 速度：几秒内完成

#### 2. 语音转录（TranscribeStage）
- 使用 Whisper 模型进行语音识别
- 支持本地（faster-whisper）和远程（HTTP API）两种模式
- 输出带时间戳的分段文本
- 准确率：中文普通话 95%+（清晰音频）

#### 3. 内容分析（AnalyzeStage）
- 使用 LLM（Ollama/Qwen）分析转录文本
- 自动生成知识大纲（标题、摘要、关键时间点）
- 提取关键词
- 输出结构化 JSON

#### 4. 截图提取（ScreenshotStage）
- 根据分析结果中的关键时间戳提取视频帧
- 保留重要的视觉信息（图表、代码、演示）
- 输出 JPG 格式截图

#### 5. 文档生成（GenerateStage）
- 使用 Jinja2 模板渲染 Markdown
- 嵌入截图
- 生成可读性强的知识文档

| **测试** | pytest + pytest-cov | 单元测试、覆盖率报告 |

### 架构特点

#### 1. 可插拔 Backend 架构

```python
# 抽象基类
class STTBackend(ABC):
    @abstractmethod
    def transcribe(self, audio_path: str) -> TranscriptResult: ...

# 实现类
class WhisperSTTBackend(STTBackend): ...      # 本地
class WhisperRemoteBackend(STTBackend): ...   # 远程
```

**优势：**
- 易于扩展新的后端（Google Cloud Speech, Azure Speech, etc.）
- 配置驱动的后端选择
- 便于测试（Mock Backend）

#### 2. Stage-based Pipeline

```python
class Stage(ABC):
    name: str

    @abstractmethod
    def run(self, context: dict) -> StageResult: ...

    def can_skip(self, tracker) -> bool: ...
```

**优势：**
- 每个阶段独立可测试
- 支持断点续处理
- 状态可追踪
- 易于添加新阶段

#### 3. 配置优先级合并

```
全局配置 (~/.vbook/config.yaml)
    ↓ 深度合并
项目配置 (./vbook.yaml)
    ↓ 覆盖
CLI 参数 (--config, --output, etc.)
```

**优势：**
- 灵活的配置管理
- 支持多环境（开发、生产）
- CLI 参数可临时覆盖

## 约束条件

### 技术约束

1. **Python 3.11+** - 使用现代 Python 特性（类型提示、dataclass）
2. **FFmpeg 依赖** - 必须预先安装
3. **GPU 推荐** - Whisper 和 LLM 在 GPU 上运行更快
4. **网络依赖** - 远程模式需要稳定的内网连接

### 业务约束

1. **CLI 优先** - 当前版本仅支持命令行，Web UI 后续考虑
2. **开源模型** - 优先支持本地部署的开源模型
3. **中文优先** - 当前主要针对中文内容优化

## 项目价值

### 对个人用户

- **节省时间** - 1 小时视频 → 5 分钟阅读

- **SEO 优化** - 文字内容更易被搜索引擎收录
